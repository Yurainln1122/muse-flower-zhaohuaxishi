# 《暮色拾花》构建说明

## 成品目标

构建一个面向中文玩家、桌面与手机浏览器均可完整操作、首次游玩约 15 分钟的单页网页游戏。玩家以原创“拾花校勘员”身份，在四处纸上记忆空间中处理 12 张片段，最终得到可失败、可修订、可成功并可重开的完整结果。

## 必读设计

- `analysis/SOURCE_BIBLE.md`
- `concepts/CONCEPT.md`
- `design/GAME_DESIGN.md`
- `design/ART_DIRECTION.md`

## 必须保真

1. 核心循环：翻面 → 可选查证 → 盖“留温/留刺/存疑”签 → 世界响应 → 新证回流 → 改签。
2. 玩家是原创角色，不扮演鲁迅或原作人物；空间跨篇重组，不做章节复演。
3. 至少有一次由新证触发的强制回看（隐鼠↔阿长）。
4. 范爱农死因只能“存疑”，不可生成确定答案。
5. 正确结果必须容纳温度、芒刺与不确定；偏色结果必须可返回修订。
6. 视觉为原创纸上园林、木刻线与石印套色语法；状态同时用形状、文字与颜色表达。
7. 首发界面只有简体中文；玩家文案集中、可替换，不烙进图片。

## 范围

### 必须包含

- 开场、4 个连续场所、12 张主片、夕照书案、偏色与清醒两类结果。
- 鼠标、触控和键盘操作；静音、暂停/帮助、结果后修订与完全重开。
- 夕光 6 格；查证和首次误签消耗，耗尽后仍可完成。
- 确定性状态；刷新前无需持久化。
- `1366×768` 与 `390×844` 可读可操作。

### 明确排除

- 登录、后端、云存档、排行榜、多人、开放世界、战斗、全文阅读、配音、多语言与外部图片依赖。

## 实现自由

根据当前环境自行选择最能实现已批准设计的技术方案。优先保证完整循环、响应式可读性和可测试状态；视觉资产允许由 CSS、HTML、Canvas 与程序化形状完成，但不要生成或内联 SVG 插画。

## 完成证据

- 可重复的本地运行命令与构建命令。
- 页面加载、无关键控制台错误、真实输入导致画面与状态变化。
- 隐鼠回流、三签反馈、夕光消耗、偏色结果、返回修订、清醒结果和重开均实际走通。
- 桌面与手机视口截图。
- QA 报告中零 `blocker`、零 `major`。

## 构建后记录

- 实现：Next.js 兼容单页客户端游戏，使用 vinext/Vite 构建；无后端、登录或外部图片依赖。
- 本地命令：进入 `build/app` 后执行 `pnpm install`、`pnpm dev`；生产验证执行 `pnpm build`。
- 自动验证：生产构建成功；服务端渲染与 12 残片/结果态测试 2/2 通过；ESLint 零问题。
- 浏览器验证：真实点击完成翻证、细察、误判、改签、隐鼠证据回流、四景推进、偏色结果、返回修订、清醒结果和完全重开。
- 响应式验证：桌面默认视口和 `390×844` 触屏视口均可读可操作；手机页面无横向溢出。
- 已知限制：本作按批准范围不保存刷新前进度；提示音由浏览器 Web Audio 即时生成，静音可随时切换。
